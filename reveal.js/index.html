What is a bug?

Good Habits for Avoiding Bugs
  • Read your code!
  • Write short bits of code at a time & run the code frequently
  • Try bits of code in Python shell first
  • Plan your code before you start writing
  • Close things as soon as you open them
  • Pair programming!
  • Organization: Classes and methods should each have a single responsibility
  • Don't repeat yourself (DRY)
(More) Good Habits for Avoiding Bugs
  • Use a good text editor & use its shortcuts
  • Using meaningful names (for variables, functions, classes)
  • Be consistent
  • Comment your code
  • Code review
  • Read the docs
  • Use best practices (eg. avoid global variables)
Debugging Techniques
  • Read the error messages
    • message (all of it!)
    • line number
    • stack trace
  • Read the code
  • Print things to the screen
  • Simplifying your code
(More) Debugging Techniques
  • Read the docs
  • Experiment in Python shell
  • Rubber duck debugging: explain the problem to someone else
  • Google / Stack Overflow
  • Use a debugger to set breakpoints
