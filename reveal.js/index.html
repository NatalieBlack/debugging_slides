What is a bug?

Good Habits for Avoiding Bugs
  • Read your code!
  • Write short bits of code at a time & run the code frequently
  • Try bits of code in IRB first
  • Plan your code before you start writing
  • Close things as soon as you open them
  • Pair programming!
  • Organization: Classes and methods should each have a single responsibility
  • Don't repeat yourself (DRY)
(More) Good Habits for Avoiding Bugs
  • Use a good text editor & use its shortcuts
  • Proper indentation
  • Using meaningful names (for variables, methods, classes)
  • Be consistent
  • Comment your code
  • Code review
  • Read the docs
  • Use best practices (eg. avoid global variables)
  • Testing!
Debugging Techniques
  • Read the error messages
    • message (all of it!)
    • line number
    • stack trace
  • Read the code
  • Print things to the screen
  • Simplifying your code
(More) Debugging Techniques
  • Read the docs
  • Experiment in IRB
  • Testing
  • Rubber duck debugging: explain the problem to someone else
  • Google / Stack Overflow
  • Use a debugger to set breakpoints

  • syntax error, unexpected keyword_end, expecting end-of-input

    You have an extra end somewhere (could be anywhere) in your file

  • syntax error, unexpected end-of-input, expecting keyword_end (SyntaxError)

    You are missing an end somewhere (could be anywhere) in your file

wrong number of arguments (given _, expected _) (ArgumentError)

  • It could be that you're passing the wrong number of arguments

  • It could be that you didn't set up the arguments correctly in the method definition

undefined method `_' for nil:NilClass (NoMethodError)

What are you trying to call that method on? The problem isn't the method - it's that the variable you're calling it on contains nil. Look into where that variable got defined/assigned.

undefined method `____' for ____ (NoMethodError)

  • It could be that you are trying to call a method that doesn't exist.

  • It could be that the thing you're calling the method on isn't the data type you think it is.

  • It could be that you can't use that data type the way you think you can

undefined local variable or method `_' for _ (NameError)

  • It could be that you didn't define that method/variable

  • It could be that you have a typo

  • It could be that you don't have access to that method/variable in the place you're trying to use it
